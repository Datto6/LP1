$1.Definicao:@ Uma abstracao de codigo, que representa uma computacao feita com n passos definidos.~
$2.Vantagens:@ Modularizacao do codigo, + legivel, mais facil de achar erros(manutencao), e sem reusar linhas~
$3.Desvantagens:@ Se perder em funcoes e abstracoes infinitas tb eh complicado ~
$4.Funcoes Importantes(EM GERAL):@ 
->void* malloc(size_t size)
->void free(void * ptr)
-> size_t strlen(const char* str)
->strncpy(char * destination, char * source, size_t n)
->assert( expressao logica)~
$5.Funcoes Importantes(ARQUIVOS):@ 
->FILE *fopen(const char *restrict path, const char *restrict mode)
-> size_t fread(&entrada, size_t tamanho, size_t numero, FILE * arquivo)
->size_t fwrite(&entrada,size_t tamanho,size_t numero, FILE * arquivo)
->fputc:@ int fputc(int char, FILE *pointer)
->~
$6.Printf e scanf(funcoes IO):@ 
->"%d" "%c" "%x" "%hhd" "%s" == int, letra, hexadecimal, 1 byte de int, string respectivamente
-> puts(str): igual o printf, so que bota o \n automatico la
->~
